\section{Clustering}
This class is a type of \textit{unsupervised learning} in which we assume that the 
data has some kind of unknown structure, and we want to figure it out. To do that we
try to accomplish two things: 
\begin{itemize}
    \item minimize distance between datapoints of the same cluster
    \item maximize distance between \diff clusters
\end{itemize}
\begin{figure}[h]
    \centering
    \includegraphics[width=0.3\textwidth]{clustering/cluster_example.png}    
\end{figure}
The clustering output is really subjective and depend what we're trying to achieve
and our conception of \textbf{similarity}. We have to define two elements:
\begin{description}
    \item[Distance metric]: tells what makes two object similar/dissimilar
    \item[Clustering algorithm]: procedure to compute groups/clusters
\end{description}

\subsubsection*{Distance metrics}
The definition of Distance Metrics depends mostly on the type of the data features.
Some common ones are:
\begin{itemize}
    \item \textbf{Euclidean distance}: $d_E(x,q)=\sqrt{\sum_{i=1}^n(x_1-q_1)^2}$

    \item \textbf{Squared Euclidean Distance}: $d_{E^2}(x,q)=(d_E(x,q))^2 
    \rightarrow$ with respect to $d_E$ it gives more emphasis to outliers 
    \textcolor{gray}{(distance is squared)}

    \item \textbf{Standardize Euclidean Distance}: $d_{SE}(x,q)=\sqrt{\frac{1}
    {s_i^2}(x_i-q_i)} \rightarrow$ weights each dimension by something inverse
    to its variability, favouring those dimensions that doesn't change too much

    \item \textbf{Chebyshev Distance}: $d_ {\max}(x,q)=\max_i|x_i-q_i|$ 
    \textcolor{gray}{(just the max, not really useful)}

    \item \textbf{Manhattan Distance}: $d_M(x,q)=\sum_{i=1}^n|x_i-q_i| \rightarrow$
    gives the \# step to reach a destination in a grid by moving up and down, lot of
    usage in many fields

    \item \textbf{Canberra Distance}: $d_{\text{Canberra}}(x, q) = \sum_{i=1}^{n} \frac{|x_i - q_i|}{|x_i| + |q_i|} \rightarrow$ this distance magnifies the impact of small differences when absolute values are small.

    \item \textbf{Correlation-based Distance}: $d_R(x,q)=1-\cfrac{s_{x,q}}{\sqrt{s_x}
    \sqrt{s_q}}=1-\cfrac{\sum_i(x_i-\bar{x})(q_i-\bar{q})}{\sqrt{\sum_i(x_i-\bar{x})
    ^2\sum_i(q_i-\bar{q})^2}} \rightarrow$ this measures allow to get an idea about
    the distance between feature trends distance 
\end{itemize}
\hfill \\
Highly suggested to use $d_R$ in the project\\
\hfill \\
To compare the datapoint features, we must be sure that everyone is using the same
measurement unit.

\subsubsection*{Clustering algorithms}
There are several way to categorize these methods:
\begin{description}
    \item[Hard clustering]: each datapoint can be only in one cluster
    \item[Soft clustering]: each datapoint might be in more clusters at the same time
\end{description}
\begin{figure}[h]
    \centering
    \includegraphics[width=0.6\textwidth]{clustering/soft_vs_hard_clustering.png}
\end{figure}

\begin{description}
    \item[Flat clustering]: all clusters are at the same level
    \item[Hierarchical clustering]: the cluster are under some hierarchy (dendrogram)
\end{description}

\subsection{K-means algorithm}
In this algorithm we priorly fix the number of clusters $k$, starting from an initial 
assignment and iteratively moving datapoints between the clusters.

% maybe make it like code snippet
\begin{enumerate}
    \item define $k \rightarrow C_1\dots C_k$ clusters
    \item randomly assign each point to some cluster $C_i$ and compute the centroid for 
    that set of points
    \item re-assign each point to the closest centroid cluster, ending up possibly with 
    new point in each cluster
    \item recompute the centroids, and iterate from \textit{step 2}
\end{enumerate} 
We iterate until the algorithm converges, e.g. the points won't change cluster.

\textbf{Limitations} of this algorithm:
\begin{itemize}
    \item the algorithm output strongly depends on the initial random assignment; if 
    there is some structure probably the algorithm will converge at the same result, but 
    if this structure is not so evident the outcome might be very sensitive.\newline
    \begin{figure}[h]
        \centering
        \includegraphics[width=0.2\textwidth]{clustering/kmeans/init_assignment_error.png}
    \end{figure}
    To reduce this influence we could try out multiple seeds in parallel and choose at the 
    end the best result.

    \item we have to choose $k$ in advance, maybe without clear idea of we're doing
    
    \item the distance used affects a lot the centroid: if we use \textit{Euclidean 
    Distance} our computation is going to be very sensitive to outliers
    \begin{figure}[h]
        \centering
        \includegraphics[width=0.2\textwidth]{clustering/kmeans/k_medoids.png}
    \end{figure}
    We can resolve this by choosing another method to compute distances: instead of using 
    $d_E$ we use the \textbf{mode}, which are more robust to outliers $\rightarrow$ the 
    \textit{medoid} is also more reasonable to use since we'll end up with something that 
    is a real point of the cluster \textcolor{gray}{(like the most representative point)}.
    
    \item with strange-shape clusters, K-means makes a very bad work (how can we know that 
    in advance? You can't) $\rightarrow$ k-means fits well spherical/ellipsoidal 
    datashapes
\end{itemize}

\subsection{Hierarchical clustering}
%It can be used for many many things, not only for clustering
This method builds a hierarchy among the clusters using a \textbf{Distance Matrix D}, which 
contains the distance between each pair of samples. To stop the procedure we have to define 
a \textbf{cutting criterion} to stop the construction of the \textbf{dendrogram} := 
representation of your cluster.\\
\begin{figure}[h]
    \centering
    \includegraphics[width=0.4\textwidth]{clustering/hierachical/dendogram_example.png}
\end{figure}
The algorithm can both start by considering each datapoint in a \diff cluster and then
merging them, or start from a one big cluster and progressively split it.

\begin{enumerate}
    \item start from considering each datapoints as a cluster per se
    \item merge the two closer clusters and take a "center" (linkage methods)
    \item recompute the set of points and iterate starting from 1
\end{enumerate}
By merging the cluster $D$ become smaller at each iteration.

By using this algorithm we can also highlight noisy datapoints and anomalies: if a point is 
added very late in the cluster, maybe it's just an anomaly $\rightarrow$ this method can
also be used for anomaly detection.

\paragraph{Linkage methods}\hfill\\
How do we compute the "center" between build-up clusters? 
\begin{description}
    \item[Single linkage]: cluster-distance is based on the two closest points from each 
    cluster. It tends to create long chain-like clusters. Suited for clusters naturally spread out.
    \begin{figure}[h]
        \centering
        \includegraphics[width=0.1\textwidth]{clustering/hierachical/single_linkage.png}        
    \end{figure}

    \item[Average linkage]: cluster-distance is based on the average position of the cluster
    datapoints. Less sensitive to outliers than \textit{simple linkage} but more flexible than \textit{complete linkage}
    \begin{figure}[h]
        \centering
        \includegraphics[width=0.1\textwidth]{clustering/hierachical/avg_linkage.png}        
    \end{figure}

    \item[Complete linkage]: cluster-distance is based on the most distant points from each
    cluster. It esnures tight and well-defined clusters, more resilient to outliers, but when data are naturally spread it might over-fragment them.
    \begin{figure}[h!]
        \centering
        \includegraphics[width=0.1\textwidth]{clustering/hierachical/complete_linkage.png}        
    \end{figure}

\end{description}
There is no rule of thumb, start with the average and adjust things.
% what about the mode for this one?

The \textit{cophenetic distance} between two points is the height at which they are merged in the same cluster.
The \textit{Cophenetic Correlation  Coefficient} instead is a metric to compare how well the hierarchical clustering preserves the orginale pairwise distances (basically it's the correlation between the cophenetic distance and the normal distance)
$$
CCC = \frac{\sum (d_{ij} - \bar{d})(c_{ij} - \bar{c})}{\sqrt{\sum (d_{ij} - \bar{d})^2 \sum (c_{ij} - \bar{c})^2}}
$$

\subsection{Density-based clustering, DBSCAN}
The idea is thinking about clusters as \textit{dense regions} of my data space separated by regions 
of low-density points. This method, not being based on distances, is very useful for data 
distributed following strange shapes. We also don't need to specify the number of clusters 
in advance.

\subsubsection{Density-Based Spatial Clustering of Applications with Noise (DBSCAN)} \hfill \\
This method is able to find arbitrary shaped cluster without prefixed total number, also in 
presence of noise. We can tune using two parameters:
\begin{itemize}
    \item \texttt{eps} $\varepsilon$: distance used to define neighbourhoods $\rightarrow$ used to 
    define the area on which we figure out the density
    \item \texttt{minPts}: minimum number of points to define a cluster
\end{itemize}
\hfill \\

\begin{description}

    \item[Border point]: is reachable (is in the radius) from a core point, but centering a circle
    on himself there aren't enough points to form a cluster
    \item[Noise points]: neither \textit{core point} or \textit{border point}, it's an 
    outlier
\end{description}

\subsection{Probabilistic clustering, mixture models}
Allows to be more flexible on the assignment of datapoints to clusters. We model each cluster as a 
\textit{component} that has assigns to each datapoint the relative membership probability.\\
But why do we need this? All the methods shown before don't tell you their level of uncertainty
of placing the point into the clusters. 

\paragraph{Mixture model}\hfill\\
It's a probability model $p(c)$ which relies on describing the probability using data taking into
account more sources, each one of them uses a different probability distribution $f_k(x;\theta_k)$
with some parameters set $\theta_k$ and is accounted in the global model with some weight $w_k$. \\
The weight tells how much is probable that point is generated starting from that model.
The underlying idea is to \textit{describe the population as a blend of $K$ \diff populations}. \\
For each cluster we have a component, which is a \diff source/subpopulation ecc. ecc.

% should blend the description
\begin{equation*}
    p(x) = \sum_{k=1}^K w_kf_k(x;\theta_k)
\end{equation*}
\hfill \\
In the course we will focus on the \textbf{Gaussian Mixture Model (GMM)}.
\begin{figure}[h!]
    \centering
    \includegraphics[width=0.4\textwidth]{clustering/prob/gmm.png}
\end{figure}
\begin{equation*}
    p(x) = \sum_{k=1}^K \pi_k \mathcal{N}(x|\mu_k,\Sigma_k)
\end{equation*}
where $\mu_k$:= subpopulation center, $\Sigma_k$:= subpopulation "shape" (how much and in which
direction is shaped) and $\pi_k$:= how common the subpopulation is (in other words).
\vspace{0.5cm}

We start by adding \textit{latent variables} $Z\in \{1\dots K\}$ which are some non-observable
variables indicating which component/cluster generates my datapoints. First we draw some preliminary 
shape for the components $\mathbb{P}(Z=k)=\pi_k$ \textcolor{gray}{(in other words, means giving
some probability to the whole cluster)}.
Once fixed the parameters, we try to figure out \textit{given some point $x_i$ what's the 
probability it came from component $k$?}
\begin{equation*}
    \gamma_{ik} = \prob(Z_i=k|x_i,\theta) = \cfrac{\pi_k \mathcal{N}(x_i|\mu_k,\Sigma_k)}
    {\sum_{j=1}^K\pi_k \mathcal{N}(x_i|\mu_k,\Sigma_k)}
\end{equation*}
where $\gamma_{ik}$ := \textbf{responsibilities} \textcolor{gray}{(how much the cluster is 
responsible for the creating the point)} is the metric that allows for \textbf{soft clustering}.
This is very useful when clusters overlap.

If you want to go back at \textbf{hard clustering} you can take $\hat{z}_i=\arg \max_k\gamma_{ik}$.
As in \textit{k-means} in \textit{GMM} we have to choose in advance the \# of clusters, however
thanks to soft clustering this prior choice has less impact on the final outcome.
This type of algorithm suffers in scalability (you have to tune all then parameters which are not
hyperparameters), however it gives us a lot of information about the data.

\subsubsection{Estimation-Maximization (EM) algorithm}
How do we estimate the model parameters? \arrow our goal is to maximize the log-likelihood 
$\ell(\theta)$, and we can do that by using EM-algorithm.
\begin{equation*}
    \ell(\theta) = \sum_{i=1}^{n} \log \left( \sum_{k=1}^{K} \pi_k \mathcal{N}(x_i \mid \mu_k, 
    \Sigma_k) \right)
\end{equation*}

The \textbf{EM-algorithm} works as follows:
\begin{enumerate}
    \item start by randomizing the model parameters $\theta^{(0)}$
    \item \textbf{Expectation step (E-step)}: given the actual parameters $\theta^{(t)}$, compute 
    the responsibilities $\gamma_{ik}^{(t)}$ \arrow the idea is to use the actual best-guess 
    parameters to softly-assign each point to the clusters
    \item \textbf{Maximization step (M-step)}: given the actual datapoint-cluster assignment 
    $\gamma_{ik}^{(t)}$, re-estimate the parameters $\theta^{(t)}$ \arrow for each cluster $k$:
    \begin{itemize}
        \item count (soft-count) the number of points in the cluster: $N_k^{(t)}=\sum_{i=1}^n
        \gamma_{ik}^{(t)}$
        \item compute the new "popularity" of my cluster: $\pi_k^{(t)}=\frac{n_k^{(t)}}{n}$
        \item compute the new cluster mean: $\mu_k^{(t+1)} = \frac{1}{N_k^{(t)}} \sum_{i=1}^{n} 
        \gamma_{ik}^{(t)} x_i$
        \item compute the new cluster covariance: $\Sigma_k^{(t+1)} = \frac{1}{N_k^{(t)}} \sum_
        {i=1}^{n} \gamma_{ik}^{(t)} (x_i - \mu_k^{(t+1)})(x_i - \mu_k^{(t+1)})^T$
    \end{itemize}
\end{enumerate}
So we basically try to do the assignment, update the structure of the cluster with the new 
assignments and so go on.
\begin{figure}[h!]
    \centering
    \includegraphics[width=0.4\textwidth]{clustering/prob/em_example.png}
\end{figure}

This algorithm is iterative; these two steps are repeated until convergence on $\ell(\theta)$ or
up to a certain \# of iters. One cool thing of \textit{EM-algorithm} is that is designed so The
value will never decrease \arrow converges toward to a local optimum (might get stuck in locals)

\subsubsection{Pros \& Cons}\hfill\\
\begin{itemize}
    \item[\textcolor{green}{$\bullet$}] unlike \textit{k-means}, this can adapt to many data shapes
    \item[\textcolor{green}{$\bullet$}] can adapt to cluster with varying sizes
    \item[\textcolor{green}{$\bullet$}] soft assignments are more informative to understood the 
    data than hard assignments 
    \item[\textcolor{green}{$\bullet$}] can deal with overlapping clusters
    \item[\textcolor{green}{$\bullet$}] can be used for density estimation and anomaly detection
    
    \item[\textcolor{red}{$\bullet$}] \# of cluster must be defined in advance
    \item[\textcolor{red}{$\bullet$}] Gaussian distribution might not be always the best choice 
    \textcolor{gray}{(just change the distribution)}
    \item[\textcolor{red}{$\bullet$}] might not work properly on small dataset
    \item[\textcolor{red}{$\bullet$}] initial choice of parameters might change the results
    \item[\textcolor{red}{$\bullet$}] \textit{EM-algorithm} might get stuck in local optimum and
    its convergence might be slow
    \item[\textcolor{red}{$\bullet$}] badly-conditioned covariance matrices (like almost singular) 
    might cause numeric instability
    \item[\textcolor{red}{$\bullet$}] computationally expensive
\end{itemize}

\subsection{Evaluating cluster quality}
How do we evaluate if our clusters make sense if we don't have an output reference?

\subsubsection{Within Cluster Sum of Square (WCSS)}
WCSS measures the variability of the observations within each cluster.  In other words, it measures how compact the cluster is.
\begin{equation}
    WCSS = \sum_i\sum_{p\in C_i} dist(p,C_i)^2
\end{equation}

The smaller our clusters are, the lower(better) the value of WSS. However by following
this metric, we get that the best result is we each datapoint form a cluster per-se. So we 
don't want to minimize this, otherwise is useless; we need just to find a point in which it
doesn't decreases a too much.\\
\hfill \\
With this method we're also looking at how similar points within the same cluster are, 
without considering the other goal of clustering: distance between \diff clusters.

\subsubsection{Between-clusters Sum of Square (BSS)}
BSS measures how far away the clusters are.
$$
BSS = \sum_{k=1}^{K} |C_k| \cdot \| \mu_k - \mu \|^2
$$
where $\mu$ is global avarage of all points. We would like to maximize this metric in order
to make the clusters as distinct as possible.

\subsection{Silhouette Score}
This metrics takes into account both the distance between points in the same cluster and
the distance from other cluster's points, measuring therefore both the cohesion and separation of clusters.

\begin{enumerate}
    \item $\forall$ point $i$ get all the distances from the points within the same
    cluster $a(i)$
    \item $\forall$ point $i$ get the distance from the points in the nearer cluster $b(i)$
    \item $\forall$ point $i$ get the average of both $\bar{a}(i),\bar{b}(i)$
    \item get the \textbf{Silhouette score} $\forall\ i$: $sil(i)=\cfrac{\bar{b}(i)-\bar{
    a}(i)}{\max\{\bar{a}(i),\bar{b}(i)\}}$
    \item get the \textbf{Silhouette score for cluster $k$}: $sil(C_k)=\frac{1}{|C_k|}\sum_{ 
    i\in C_k}sil(i)$
    \item get the \textbf{Silhouette score for overall clustering}: $sil(C)=\cfrac{1}{K}
    \sum_{k=1}^K sil(C_k)$
\end{enumerate}

If for some point $i$ the $sil(i)$ is +1 the point is much closer to its own cluster with respect to tother cluster, for
values near 0 the points lies in the edge between clusters and for values -1 the point probably is misclusterized.

\subsection{Model-based clustering quality}
We try to exploit the model-structure (like in \textit{probabilistic clustering}) to evaluate our 
cluster output. We have some models $M_1\dots M_R$ fitting the dataset $x_1\dots x_n$, and for
each model we have:
\begin{itemize}
    \item $\hat{\theta}$ MLE parameters
    \item $\ell(\hat{\theta})$ maximized log-likelihood
    \item $k$ number of free parameters in the model
\end{itemize}

\subsubsection{Akaike Information Criterion (AIC)}
\begin{equation*}
    AIC = 2k -2\ell(\hat{\theta})
\end{equation*}
\begin{itemize}
    \item $2k$ penalizes complexity (more parameters)
    \item $-\ell(\hat{\theta})$ rewards highly likelihood models (minimize information loss)
\end{itemize}
\arrow for each model compute the AIC and choose the smallest one.

\subsubsection{Bayes Information Criterion (BIC)}
\begin{equation*}
    BIC = k\ \log(n) -2\ell(\hat{\theta})
\end{equation*}
Same as \textit{AIC}, it penalizes model complexity and rewards high likelihood, but now the
penalization becomes stronger as the sample size increases.
